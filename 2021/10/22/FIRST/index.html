This is an example for gvRPC - java

现在我们要开发一个项目，目标是通过对服务端发起一次请求，参数使用task_name 和 token 信息，然后服务端返回 S3 的 key。

开始之前：

在开始之前需要对gvRPC的基本结构有一些简单的了解，下面是简单介绍

  • gv-common-rpc 是 gvRPC - java 的基础库，主要管理对gvRPC - java 对 gRPC 的依赖
  • spec_internalrpc_server 是gvRPC - java 的仓库，主要管理gvRPC - java 的 proto文件和生成的接口文件（即业务接口），该仓库继承 gv-common-rpc

现在我们开始：

这个项目需要实现如下功能：

服务端：

  • 继承 spec-internalrpc-server 编译的接口文件

  • 接受服务端的请求参数

  • 根据 task_name 和 token 信息 查询到对应的 S3 key 信息

  • 将S3 key信息返回到客户端

客户端：

  • 包装请求参数
  • 发送请求到服务端
  • 接受并处理服务端返回的响应数据
